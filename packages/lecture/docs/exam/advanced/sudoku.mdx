# Sūdoku

Es soll eine Sūdoku-Spiel für den Browser erstellt werden.

## Mindestanforderungen

* Auswahl zwischen 5 verschiedenen Sūdoku-Rätseln. Zudem soll es dem Spieler
  auch möglich sein, ein eigenes Sūdoku-Rätsel einzugeben. Das Format hierfür
  kann beliebig festgelegt werden, z.B. JSON. Das Format muss __kurz__
  an der Oberfläche für den Nutzer beschrieben werden. Alternativ kann auch eine
  UI für die Eingabe neuer Sudokus umgesetzt werden.
* Die 5 Sūdoku-Rätsel können fest hinterlegt werden, es ist keine automatische Generierung erforderlich.
* Nach Auswahl oder Eingabe eines eigenen Rätsels beginnt das Rätsel. Aufgabe
  des Spielers ist es, in jedes freie Feld die korrekte Zahl einzutragen.
* Die Sūdoku soll nur die Ziffern 1-9 bei der Eingabe erlauben, vorgegebene
  Ziffern dürfen nicht geändert werden.
* Nach vollständiger Eingabe aller Ziffern durch den Spieler soll die Sūdoku-App
  prüfen, ob die Lösung korrekt ist. Falls nicht, kann der Spieler seine Zahlen
  ändern und erneut prüfen lassen.
* Es soll eine digitale Uhr geben (`mm:ss`), welche die bereit benötigte
  Zeit anzeigt. Nach korrekter Lösung des Sūdoku-Rätsel soll die Uhr stoppen.
* Bei Klick auf eine Sūdoku-Feld mit bereits eingetragener Ziffer sollen alle
  Felder mit der gleichen Ziffer hervorgehoben werden (z.B. durch andere Hintergrundfarbe).

## Hinweise

Es ist nicht zwingend eine Bibliothek erforderlich. Für die Sūdoku-Felder
können auch `<input>` verwendet und entsprechend gestyled werden.

Vor Beginn sollte man sich eine geeignete Datenstruktur überlegt werden, also
wie das Sūdoku im JavaScript dargestellt und gespeichert wird.

Das Programm muss keine Sūdoku lösen können &mdash; das Überprüfen, ob eine
bestimmte Belegung der Felder gültig ist oder nicht, lässt sich wesentlich einfacher
programmieren.

Optional kann als Erweiterung auch localStorage oder IndexedDB verwendet werden,
um Daten zu speichern (z.B. High-Score, Lösungen, selber eingegebene Sūdoku).
