Abschnitt 1 - Web-APIs

Mittlerweile stellen Browser umfangreiche APIs bereit, die wir verwenden können, um verschiedenste Dinge zu tun, wie etwa auf die Webcam des Rechners zuzugreifen. In diesem Abschnitt wollen wir nicht alle APIs erlernen, sondern uns beispielhaft einige davon anschauen, um eine Vorstellung von den Möglichkeiten zu kriegen.

Eine Auflistung vieler Web-APIs finden wir bei MDN web docs - Web APIs.

Web Storage API​

Die Web Storage API erlaubt uns, Daten im Browser zu speichern. Lädt der Browser die Seite erneut oder schließt die Webseite und besucht sie später wieder, sind die Daten noch vorhanden.

Es gibt 2 Arten von Storage:

  • localStorage — Die Daten werden solange gespeichert, bis wir sie löschen (oder der Nutzer ein Löschen der Daten erzwingt.)
  • sessionStorage — Die Daten werden gespeichert, bis die Browser-Session vorbei ist. In der Regel ist die Browser-Session vorbei, wenn der Browser geschlossen wird. Das kann aber je nach Browser variieren. Einige Browser erlauben es beispielsweise, die Session beim Neustart beizubehalten.

Wir können auf die beiden Arten von Storage über die globalen Variablen localStorage und sessionStorage zugreifen.

Um Daten zu speichern, benutzen wir die setItem-Methode. Key und Value müssen jeweils ein String sein:

localStorage.setItem("key", "value");
sessionStorage.setItem("key", "value");

Zum Auslesen gespeicherter Daten steht uns die getItem-Methode zur Verfügung.

// Gibt jeweils "value" zurück
localStorage.getItem("key");
sessionStorage.getItem("key");

Hier ist ein Beispiel, wo wir den Wert einer Textarea im localStorage speichern. Beim Neuladen der Seite geht der Wert nicht verloren.

Geolocation API​

Die Geolocation-API ermöglicht uns, auf den aktuellen Standort des Nutzers zuzugreifen. In Kombination mit einem Karten-Widget können wir das etwa nutzen, um den Standort des Nutzers anzuzeigen. Aus Datenschutzgründen muss der Nutzer dem zustimmen, bevor wir im JavaScript seinen Standort erhalten können.

Im JavaScript können wir auf die Geolocation-API über das globale Objekt navigator.geolocation zugreifen:

// Wir müssen ein Callback übergeben, was aufgerufen wird, wenn
// die Position des Nutzers bekannt ist.
navigator.geolocation.getCurrentPosition(onSuccess, onError);
function onSuccess(position) {
  console.log("Breitengrad", position.coords.latitude);
  console.log("Längengrad", position.coords.longitude);
}
function onError(error) {
  console.log("Code", error.code);
  console.log("Nachricht", error.message);
}

Media Capture and Streams API​

Die Media Capture and Streams API erlaubt uns, auf Audio- und Video-Eingaben des Rechners des Nutzers zuzugreifen. Ebenfalls aus Datenschutzgründen muss der Nutzer dem vorher zustimmen.

Audioeingaben können etwa aus einem Mikrofon kommen. Videoeingabe sind beispielsweise eine Webcam oder der Inhalt des Bildschirms (Screen Capture).

Um auf Eingabegeräte (Media Devices) zuzugreifen, steht uns das globale Object navigator.mediaDevices zur Verfügung. Dieses besitzt im Wesentlichen zwei Methoden:

  • getUserMedia — Zugriff auf Webcam oder Mikrofon
  • getDisplayMedia — Zugriff auf den Inhalt des Bildschirms. Fragt den Nutzer danach, was er freigeben möchte (Browser-Tab, Fenster, gesamten Bildschirm).

Bei beiden Methoden müssen wir ein Objekt übergeben. Dieses beschreibt, welche Anforderung der Media Stream haben muss (z.B. Auflösung oder Bildwiederholungsrate).

Notification API​

Mit der Notification API können wir dem Nutzer Benachrichtigungen anzeigen. Etwa könnte ein Chat-App Notifications verwenden, um den Nutzer über neue Nachrichten zu informieren.

Mit der Funktion Notification.requestPermission() bitten wir um Erlaubnis, dem Nutzer Benachrichtigungen senden zu dürfen. Mit Notification.permission können wir prüfen, ob der Nutzer uns bereits die Erlaubnis erteilt hat (in dem Fall brauchen wir nicht noch einmal zu fragen). Browser-Support​

Nicht jedes neue Feature wird von allen Browsern unterstützt. Wir sollten immer vorher prüfen, welche Browser in welchen Version ein Feature unterstützen und welche Browser wir unterstützen wollen.

@mdn/browser-compat-data stellt umfangreiche Daten bereit, welche Browser welche Features unterstützen. Diese Daten können wir entweder auf den entsprechenden Seiten von MDN web docs oder auf Can I Use einsehen und durchsuchen.
