Abschnitt 2 - Weitere Features

In diesem Abschnitt wollen wir uns weitere Möglichkeiten anschauen, die CSS uns bietet.

Pseudo-Klassen​

Klassenselektoren wie .required kennen wir bereits. Damit können wir Elemente anhand eines ihrer Klassennamen selektieren. Klassen können wir als eine Art Merkmal auffassen, dass einige Elemente haben und andere nicht. Etwa drückt die Klasse required aus, ob ein Eingabefeld ein Pflichtfeld ist; oder die Klasse primary-button, ob ein Button eine Primäraktion ist (Absenden, Bestellen).

Wer das Web verwendet, hat bestimmt schon einmal gesehen, wie ein Button beim Überfahren mit der Maus seine Farbe verändert hat. Maus-über-Element ist auch ein Merkmal. Um mit CSS auf diese Merkmale zu reagieren, gibt es die sogenannten Pseudo-Klassen. Diese schreiben wir immer mit einem Doppelpunkt vor dem Namen.

Andere bekannte Pseudo-Klassen sind:

  • :active - Element, das aktiv ist, etwa ein Button, der gerade geklickt wurde.
  • :focus - Element, das fokusiert ist, etwa das Texteingabefeld, in dem sich der Cursor gerade befindet.
  • :visited - Links (<a>), die schon besucht wurden.
  • :checked - Eine Checkbox, die angehakt ist.
  • :enabled / :disabled Ein Eingabefeld wie input, welches bearbeitbar oder gesperrt ist.

Zudem gibt es noch Pseudoklassen, welche die HTML-Struktur betreffen.

  • :first-child :last-child / :nth-child(n)
    • Das erste oder letzte Kindelement. :nth-child(1) ist das 1. Kindelement, :nth-child(2) das zweite Kindelement usw.
  • :first-of-type / :last-of-type / :nth-of-type()
    • Analog, sucht aber nach dem ersten oder letzten Element eines Typs innerhalb der Geschwisternelemente. input:first-of-type such also nach dem ersten Element vom Typ input.

Beispiele für :*-child und :*-of-type. Nehmen wir an, wir haben folgendes HTML:

<div>
  <h1>Überschrift</h1>
  <p>Erster Absatz</p>
  <p>Zweiter Absatz</p>
  <p>Dritter Absatz</p>
  <h2>Unterüberschrift</h2>
</div>

Dann können wir folgendes selektieren:

div > :first-child {
  /* <h1>Überschrift</h1> */
}

div > nth-child(3) {
  /* <p>Zweiter Absatz</p> */
}

div > :last-child {
  /* <h2>Unterüberschrift</h2> */
}

p::first-of-type {
  /* <p>Erster Absatz</p> */
}

p::last-of-type {
  /* <p>Dritter Absatz</p> */
}

/* Selektiert die 3 Elemente h1, p, h2 */
/* Diese sind jeweils die ersten ihres Typs */
div > ::first-of-type {
  /* <h1>Überschrift</h1> */
  /* <p>Erster Absatz</p> */
  /* <h2>Unterüberschrift</h2> */
}

:nth-child kann auch in der Form :nth-child(2n) verwendet werden, um jedes 2. Element zu selektieren. Das können wir etwa benutzen, um jede 2. Zeile einer Tabelle andersfarbig darzustellen:

Schließlich gibt es noch :not(input), um ein Element zu selektieren, welches kein input ist. Innerhalb von not kann ein beliebiger Selektor stehen.

Mit :has können wir ein Element selektieren, das bestimmte Kinderelemente hat, z.B.

/* <div><label></label><input /></div> */
div:has(input) {
  /* ... */
}

Noch mehr Pseudo-Klassen finden sich bei MDN web docs - Pseudo Classes.

Pseudo-Elemente​

Analog zu Pseudo-Klasesn gibt es auch noch sogenannte Pseudo-Elemente. Damit können wir Elemente selektieren, die man im Browser zwar sehen kann, die aber keine direkte Entsprechung im DOM haben.

Pseudo-Elemente schreiben wir immer mit 2 Doppelpunkten am Anfang.

  • ::first-letter - Das erste Zeichen des Textinhalts eines Elements.
  • ::first-line - Das erste Zeile des Textinhalts eines Elements.
  • ::placeholder - Der Platzhaltertext in einem Texteingabefeld.

Custom-Properties (Variablen)​

Bisher haben wir bei allen Eigenschaften wie color oder font-size immer nur feste Werte verwendet.

Ein häufiger Anwendungsfall bei Webseiten ist, dass die gesamte Webseite im Style der Firma (Style Guide) gestaltet wird. Dazu gehört etwa auch, dass wir überhaupt gleiche Schriftarten, Schriftgrößen und Farben verwenden.

Natürlich könnten wir manuell darauf achten, das immer zu tun. Aber wenn etwa eine Farbe geändert werden soll, müssen wir überall in allen CSS-Dateien die Farbe ersetzen-

Wäre es nicht besser, solche Werte nur einmal festlegen zu müssen? Dies können wir durch Custom-Properties erreichen, oft auch CSS-Variablen genannt.

Zuerst müssen wir die Variable definieren. Das kann in jedem Element geschehen, meist geschieht das aber an der Pseudo-Klasse :root, welche sich auf das Wurzelelement (<html>) bezieht:

:root {
  --base-font-size: 12px;
  --base-font: "Liberation Serif";
  --theme-color: orange;
}

Anschließend können wir diese Variablen via var(--theme-color) überall dort verwenden, wo wir auch sonst Farben oder Größen verwendet hätten.

Berechnungen​

Eine weitere Möglichkeit, Größenangaben dynamisch zu gestalten, ist die calc-Funktion.

Damit können wir arithmetische Berechnung durchführen, wobei die Einheiten verschieden sein können.

Zum Beispiel können wir damit eine kleine Anzahl an Pixel zur aktuellen Schriftgröße addieren. Dies stellt im Vergleich zu 1.1em sicher, dass das Ergebnis eine Ganzzahl ist:

::first-letter {
  width: calc(1em + 2px);
}
``

Beachten müssen wir, vor und nach Operatoren wie `+` immer ein Leerzeichen zu
setzten, sonst funktioniert die Berechnung nicht.

Oder wir können können die Breite auf 500px setzen, es sei denn, der Bildschirm
ist nicht breit genug:

```css
dialog {
  width: calc(min(500px, 90vw));  
}

}
