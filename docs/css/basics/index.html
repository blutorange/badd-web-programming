Abschnitt 1 - Grundlagen

Zuerst müssen wir uns anschauen, wie CSS überhaupt aufgebaut ist und welche wesentlichen Konzepte dabei eine Rolle spielen. Wir beginnen mit dem groben Aufbau und der Syntax von CSS.

Syntax​

Wir wissen schon, dass HTML aus geschachtelten Elementen besteht. Jedes Element wird vom Browser in einer bestimmten Art und Weise dargestellt. CSS ist nun eine Möglichkeit, um die Darstellung dieser Elemente zu beeinflussen. Um das zu ermöglichen, braucht es zweierlei Dinge:

  • Auswahl des oder der HTML-Elemente, deren Styling wir ändern wollen.
  • Definition des Stylings, das wir auf das Element anwenden wollen.

Selektoren​

Der obere Teil ist verantwortlich für die Auswahl der Elemente, die gestyled werden sollen. Es handelt sich dabei um einen sogenannten CSS-Selektor. Ein CSS-Selektor besteht aus Regeln, anhand derer nach HTML-Elementen gesucht werden soll.

Es gibt zwei Arten von Regeln. Zum einen gibt es Regeln, um anhand der Eigenschaften eines Elements zu selektieren:

SelektorBedeutung
*Jedes Element, unabhängig seiner Eigenschaften
inputElemente vom Typ input
[type="checkbox"]Elemente, deren Attribut type den Wert checkbox hat
.requiredElemente, welche die Klasse required besitzen
#emailElement, dessen ID gleich email ist

Zum Zweiten gibt es Regeln, um anhand der Beziehung zwischen Elementen zu selektieren. Dies erreichen wir, indem wir Selektoren kombinieren:

SelektorBedeutung
fieldset.fancyUND-Kombination. Ein Element vom Typ fieldset, was zudem die Klasse fancy hat.
fieldset inputEin input, der sich irgendwo in einem fieldset befindet
fieldset > inputEin input, der ein direktes Kindelement von fieldset ist
label + inputEin input, der direkt auf ein label folgt
label ~ inputEin input, der sich nach einem Label label befindet
fieldset, inputODER-Kombination. Alle fieldset und alle input Elemente (unabhängig von deren Beziehung)

Damit wir uns das besser vorstellen können, hier noch beispielhaft HTML-Schnipsel für die jeweiligen Selektoren:

<!-- input -->
<input />

<!-- [type="checkbox"] -->
<input type="checkbox" />

<!-- .required -->
<input class="stylish required" />

<!-- #email -->
<input id="email" />

<!-- fieldset input -->
<fieldset>
  <div>
    <input />
  </div>
</fieldset>

<!-- fieldset > input -->
<fieldset>
  <input />
</fieldset>

<!-- label + input -->
<div>
  <label></label>
  <input />
</div>

<!-- label ~ input -->
<div>
  <label></label>
  <p></p>
  <input />
</div>

Properties​

Damit wir das Styling eines Elements beeinflussen können, besitzt jedes Element sogenannte Properties oder Eigenschaften. Jede Eigenschaft steuert einen bestimmten Aspekt, wie das Element dargestellt wird, wie etwa Farbe, Schriftart und Schriftgröße oder Abstände.

Jedes Element hat durch den Standard und den Browser bereits Default-Werte für alle Eigenschaften. Mittels CSS können wir diejenigen Eigenschaften verändern, die wir für unser gewünschtes Styling benötigen.

Nachdem wie Elemente selektiert haben, geben wir eine Liste von Eigenschaften an, die wir auf einen anderen Wert setzen wollen. Hier ein Beispiel mit einigen häufig verwendeten Properties:

Je nach Zählweise kann man über 1700 verschiedene CSS-Properties zählen. Eine Erklärung all dieser Eigenschaften an dieser Stelle ist weder sinnvoll noch praktikabel. Ich empfehle, während der Arbeit an und mit Webseiten nach entsprechenden Eigenschaften zu recherchieren, wenn diese benötigt werden. Die MDN web docs sind eine von vielen guten Anlaufstellen:

https://developer.mozilla.org/en-US/docs/Web/CSS/color

Ein Merkmal von CSS-Eigenschaften sollten wir uns aber noch anschauen: Shorthand-Properties. Beispielsweise können wir im CSS den oberen, rechten, unteren und linken Abstand eines Elements separat ändern:

div {
  padding-top: 1px;
  padding-right: 2px;
  padding-bottom: 3px;
  padding-left: 4px;
}

Das funktioniert, ist aber umständlich. Alternativ können wir diese 4 Eigenschaften auch in eine einzige kombinieren, dies nennt man Shorthand:

div {
  /* top right bottom left */
  padding-top: 1px 2px 3px 4px;
}

Sind die Werte zudem auch noch gleich, brauchen wir diese nur einmal schreiben:

div {
  /* top=bottom=1px, right=left=2px */
  padding-top: 1px 2px;
}

div {
  /* top=right=bottom=left=1px */
  padding-top: 1px;
}

Nesting​

warnung

Das CSS-Nesting-Module ist noch relativ neu. Vor Verwendung sollte geprüft werden, ob alle Zielbrowser dieses unterstützen beziehungsweise die Verschachtelung durch einen Nachbearbeitungsschritt entfernt werden.

Nehmen wir an, wir haben in unserer Webseite einen Abschnitt zu Kontaktdaten und wollen in diesem Eingabefelder und Labels speziell stylen, anders als im Rest der Webseite. Dann können wir dies wie folgt tun:

fieldset.contact-details input {
  padding-left: 0.5em; 
}

fieldset.contact-details label {
  font-size: 1.2rem;
}

Das ist vom Code her etwas unschön, da wir den Selektor fieldset.contact-details mehrfach wiederholen müssen. Um Redundanz im Code zu vermeiden, können wir das kompakter schreiben, indem wir die Selektoren schachteln:

fieldset.contact-details {
  input {
    padding-left: 0.5em; 
  }
  label {
    font-size: 1.2rem;
  }
}

Beim Schachteln gibt es noch ein spezielles Symbol &, das auf das Elternelement matched:

fieldset {
  /* <fieldset class="contact-details"></fieldset> */
  &.contact-details {

  }

  /* <fieldset class="payment-details"></fieldset> */
  &.payment-details {

  }
}

At-Rules​

Bisher haben wir Selektoren kennengelernt, die immer angewandt wurden. Manchmal möchten wir aber Selektoren und Regeln auf einen bestimmten Kontext beschränken. Beispielsweise möchten wir bestimmte Regeln nur auf mobilen Geräten mit geringer Auflösung anwenden.

CSS definiert noch sogenannte @-Regeln. Diese haben allgemein Auswirkung darauf, wie das CSS verarbeitet und angewandt wird.

Diese Regeln werden mit einem @ am Anfang geschrieben, daher der Name. Bekannte @-Regeln sind etwa Media-Queries (@media), Container-Queries (@container) oder Import-Rules (@import). Diese werden wir uns in einem der folgenden Abschnitte genauer anschauen. An dieser Stelle nur ein Beispiel für die Form einer solchen Regel:

/** Beim Drucken einer Webseite über den Browser alle Farben auf Schwarz setzen */
@media print {
  * {
    color: black !important;
  }
}

Vererbung​

Wir haben gesehen, dass wir das Styling der Eigenschaften von HTML-Elementen beeinflussen können. Einen Effekt hatten wir bisher noch nicht genauer betrachtet. Wie beeinflussen Eigenschaften eines HTML-Elements die Eigenschaften der Kinderelemente?

Im CSS gibt es einige Eigenschaften, die vererbt werden.

Um das zu illustrieren, schauen wir uns folgendes Beispiel an:

Obwohl wir nur auf dem <fieldset>-Element die Eigenschaft color: darkred gesetzt haben, ist der gesamte Inhalt im Fieldset rot.

Das liegt daran, dass color eine vererbbare Eigenschaft ist. Vererbbare Eigenschaften werden immer automatisch auf alle Kinderelemente angewandt. Der geerbte Wert wird aber nur verwendet, solange am Kindelement nicht explizit ein anderer Wert gesetzt wurde:

In Sandbox öffnen

Ein typischer Anwendungsfall wäre, wenn wir eine Standard-Schriftart und Größe für die gesamte Seite setzen möchten. Auch font-size und font-family sind vererbbare Eigenschaften:

html {
  font-size: 14pt;
  font-family: serif;
}

Ob eine Eigenschaft vererbbar ist oder nicht, können wir beispielsweise definiert MDN web docs entnehmen:

https://developer.mozilla.org/en-US/docs/Web/CSS/font-size#formal_definition

Beispiele für nicht vererbbare Eigenschaften sind etwa padding (Abstand) oder border (Umrandung).

Kaskadierung​

Bisher haben wir uns auf einfache CSS-Dateien mit wenigen Regeln beschränkt. In der Praxis auf Webseiten mit vielen CSS-Regeln passiert es aber nicht selten, dass verschiedene CSS-Regeln in Konflikt miteinander stehen:

Wie werden diese Konflikte aufgelöst?

Im Folgenden werden wir die Konfliktlösung etwas vereinfacht darstellen. Alle Details finden sich bei MDN web docs - Cascade oder in CSS Cascading and Inheritance Level 4, § 6. Cascading.

Im Wesentlichen besteht die Konfliktlösung aus 3 Schritten:

  1. Es gewinnt die CSS-Regel, die aus dem Ursprung mit höchster Priorität stammt.
  2. Bei gleichen Ursprung: Es gewinnt die CSS-Regel mit der höchsten Spezifität.
  3. Bei gleicher Spezifität: Es gewinnt die CSS-Regel, die in der CSS-Datei zuletzt steht.

Ursprung

Mögliche Ursprünge einer CSS-Regel sind (von höchster zu niedrigster Priorität):

  1. User-Agent-CSS mit !important.
  2. User-CSS mit !important.
  3. Autor-CSS mit !important.
  4. Autor-CSS
  5. User-CSS
  6. User-Agent-CSS

Dabei bedeutet:

  • Autor-CSS, welches vom Autor der Webseite, also uns Programmierern stammt.
  • User-CSS, was der Nutzer hinterlegt hat , um CSS auf der Webseite zu überschreiben. Tritt eher selten auf.
  • User-Agent-CSS: CSS, welches der Browser standardmäßig mit ausliefert. Die meisten Browser bringen grundlegendes CSS mit, damit etwa Listen oder Fieldsets ein grundlegendes Styling haben.

Bei dem Autor-CSS ist nochmal zu unterscheiden zwischen CSS in <style>-Tags und externen CSS-Dateien sowie CSS im <div style="...">-Attribut von HTML-Element. Letzteres nennt man Inline-Styles, diese haben höhere Priorität.

Bei der Entwicklung von Webseiten haben wir es häufig nur mit einem Ursprung zu tun, dem Autor-CSS von uns.

!important ist ein Flag, welches wir an jeder CSS-Eigenschaft setzen können, um dieser (gemäß obiger Regeln) mehr Gewicht zu verleihen:

.hidden {
  display: none !important;
}
info

In einigen Fällen kann die Verwendung von !important durchaus sinnvoll sein, um andere CSS-Regeln zu überschreiben. Meistens sollten wir aber darauf verzichten, unser CSS noch einmal anschauen und ein bessere Lösung finden. Kurzfristig können wir mit !important zwar dafür sorgen, dass unsere CSS-Regeln angewandt wird, langfristig stoßen wir aber erneut auf Konflikte, wenn viele CSS-Regeln !important verwenden. Ein !very-important gibt es nicht.

Spezifität

Weisen 2 CSS-Regeln den gleichen Ursprung auf, wird die Spezifität der Selektoren miteinander verglichen.

Die Spezifität eines Selektors ist ein Tripel von positiven Ganzzahlen und ergibt sich aus den darin verwendeten Bestandteilen.

Der Vergleich von 2 Spezifität-Werten geschieht von links nach rechts. Sind (x1, x2, x3) und (y1, y2, y3) 2 Spezifität-Werte, wird zuerst x1 und y1 verglichen. Es gewinnt die Spezifität mit dem höchsten Werte. Sind beide Werte gleich, wird x2 mit y2 verglichen und es gewinnt der höchsten Wert. Sind diese ebenfalls gleich, wird x3 mit y3 verglichen. Beispiel:

  • (1, 1000, 20) < (2, 0, 0), weil 1 < 2
  • (9, 7, 3) > (9, 7, 2), weil 3 > 2

Die 3 Werte (x1, x2, x3) des Spezifität-Tripel ergeben sich wie folgt:

  • x1 - Anzahl der ID-Selektoren, z.B. #blue-sky
  • x2 - Anzahl der Klassenselektoren (.green-tea) und Attributsselektoren ([type="input"]).
  • x3 - Anzahl der Elementselektoren, z.B. p oder input

Es gibt bei der Berechnung der Spezifität noch einige Details zu beachten, auf die wir hier verzichten und die bei MDN web docs - Specificity nachgelesen werden können.

Kehren wir noch einmal zurück zu unseren anfänglichen Beispiel:

/* Spezifität (1, 0, 0) */
#blue-sky {
  color: blue;
}

/* Spezifität (0, 1, 0) */
.green-tea {
  color: green;
}

/* Spezifität (0, 0, 1) */
p {
  color: red;
}

Es gewinnt daher der erste ID-Selektor und das Element erhält die Farbe Blau.

Das Anwenden dieser Regeln nennt man Kaskadierung. Kaskade stammt ab vom lateinischen casicare "Fallen" und bezeichnet einen stufenförmigen Wasserfall.

Bildlich können wir uns es vielleicht so vorstellen, dass die Regeln, von höchster zu niedrigster Priorität von oben nach unten herunterfallen. Regeln niedrigerer Priorität werden somit von denen höherer Priorität verdrängt / fortgestoßen.

Konfliktlösung​

Wie wir im obigen Abschnitt sahen, sind die Regeln für die Kaskadierung teilweise recht komplex. Besser ist es in der Praxis, Konflikte soweit möglich zu vermeiden.

Ein Ansatz, der sich bewährt hat, besteht darin, soweit möglich nur Klassen in den Selektoren zu verwenden. Halten wir uns dann noch an ein Namensschema für diese Klassen, erleichtert und dies die Entwicklungsarbeit sehr.

Ein solches Namensschema ist BEM. BEM steht für Block-Element-Modifier. Ein Block ist etwa ein Dropdown-Widget. Dieses erhält die CSS-Klasse stylish-dropdown. Die Bestandteile des Dropdowns werden mit 2 Unterstrichen abgetrennt, z.B. stylish-dropdown__option-container oder stylish-dropdown__selected-text. Um ein Element oder Block anders darzustellen, wird der Modifier mit 2 Bindestrichen angehangen, z.B. stylish-dropdown--large oder stylish-dropdown__option-container--hover.

Auf dieser Art und Weise entstehen eindeutige Klassennamen, die in keine Konflikte mit anderen Selektoren geraten. Da es sich nur um eine einzelne Klasse handelt, wird auch die Spezifität gering gehalten.

Weitere Ansätze zur Konfliktlösung sind etwa verschiedene CSS-in-JS-Lösungen, auf die wir hier aber nicht näher eingehen.

Farben und Einheiten​

CSS beschreibt das Layout und Design. Dazu gehören auch Farben- und Größenangaben.

Farben

Farben können wir im CSS auf verschiedene Weisen angeben:

  • rgb(255, 0, 0) - Angabe der Rot-, Grün- und Blau-Komponente vom RGB-Farbraum.
  • #ff0000 - Auch RGB, aber Darstellung im Hexadezimalformat.
  • hsl(270deg 100% 50%) - Angabe von Hue (Farbton), Saturation (Sättigung) und Lightness (Helligkeit).

Weitere Farbräume sind etwa noch hwb, lch, lab, oklab oder oklch.

Zudem gibt es noch einige Farbnamen wie yellow oder darkorange, die wir verwenden können. Diese sind für Testzwecke hilfreich, in der Praxis ist aber die Verwendung eines konkreten Farbcodes zu empfehlen.

Falls wir Transparenz möchten, kann noch ein vierter Wert zwischen 0 (transparent / durchsichtig) und 1 (opaque / undurchsichtig) ergänzt werden:

Größenangaben

Größenangaben benötigen wir etwa für Schriftgrößen und Abstände. CSS stellt eine umfangreiche Menge von Einheiten zur Verfügung, um Größen anzugeben. Einige wichtige sind:

  • px - Angabe in Pixeln. Browser auf mobile Geräte mit hochauflösenden Displays setzen meist 1 CSS Pixel auf 2 oder mehr physische Pixel, damit Inhalte auf Webseiten nicht zu klein sind.
  • pt - Typografischer Punkt, etwas größer als 1 Pixel.
  • em - Relativ zur Schriftgröße des aktuellen Elements. Wenn das Element aktuell etwa Schriftgröße 12px hat, ist 1em = 1px.
  • rem - Ähnlich wie em, aber relative zur Schriftgröße des Wurzel-HTML-Elements.
  • % - Relativ zu einem Bezugsmaß, das von der konkreten Verwendung abhängt. Beispielsweise bedeutet width: 50% halb so breit wie das Elternelement.
  • vw - Steht für Viewport Width. 100vw ist die Breite des Browser-Fensters. Kann z.B. nützlich sein, um die Maximalbreite eines Overlay-Dialogs zu setzen.
  • vh - Viewport Height, 100vh ist die Höhe des Browser-Fensters.
  • mm / cm - Steht für Millimeter und Centimeter, ist aber nicht zwangsläufig auch eine Millimeter oder Zentimeter. Auf Computerbildschirmen kann die Größe anders sein. Beim Ausdrucken sollte mm und cm das korrekte Maß haben.

Siehe auch W3C - CSS tips & tricks.

px ist oft eine gute Wahl für Größenangaben und garantiert, dass Linien mit einem Vielfachen von 1px auch scharf dargestellt werden.

Für Schriftgrößen ist em empfehlenswert. Der Browser des Geräts, wo die Webseite dargestellt wird, kann die Schriftgröße vom html-Element entsprechend der Geräteeinstellungen setzen. Sehbehinderte Nutzer könnten diese etwa höher setzen. Alle Texte, die em verwenden, sind dann relativ zu dieser Größe. Aus dem gleichen Grund kann auch rem sinnvoll sein, was immer relativ zur Schriftgröße vom html-Element ist.

Box-Model​

Das Box-Model ist ein weiteres grundlegendes Konzept von CSS. Jedes Element wird dabei als Box betrachtet, die 4 Eigenschaften hat:

  • Content: Der Inhalt des Elements selber.
  • Padding: Innerer Abstand um das Element herum.
  • Border: Umrandung um das Element.
  • Margin: Äußerer Abstand um das Element herum.

Padding ist innerhalb der Border, Margin außerhalb der Border.

In anderen Worten, Padding wird benutzt, um den Inhalt eines Elements einzurücken, Margin wird verwendet, um ein Element von anderen Elementen zu trennen.

Die Eigenschaften width und height geben die Größe eines Elements an. Standardmäßig beziehen sich diese nur auf den Inhalt ohne die Umrandung. Durch Setzen von box-sizing: border-box können wir erreichen, dass diese beiden Eigenschaften die Größe inklusive Rand verändern. Das ist oft nützlich, wenn wir die Gesamtgröße setzen wollen und sich die Größe der Umrandung (Border) der Gesamtgröße nicht ändern soll.
