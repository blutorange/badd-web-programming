Abschnitt 4 - At-Rules

@import​

Mittels @import können wir andere CSS-Dateien inkludieren. Das ist hilfreich, um Code in mehrere Dateien aufzusplitten.

Relative URLs werden dabei immer relativ zur URL der CSS-Datei aufgelöst, in der sich der @import befindet. Im folgenden Beispiel wird das CSS der Bibliothek Bootstrap importiert, welche grundlegendes CSs enthält, um Webseiten schön aussehen zu lassen.

@import verhält sich so, als ob das importierte CSS direkt an dieser Stelle stehen würde. Wenn wir 2 CSS-Dateien a.css und b.css haben, die beide jeweils bootstrap.css importieren und dann in einer anderen Datei a.css und b.css importieren, haben wir bootstrap.css doppelt importiert.

@font-face​

@font-face ist eine Regel, mit der wir eigene Schriftarten in eine Webseiten einbetten können. Meist hat den Nutzer verschiedene Schriftarten auf seinem Rechner installiert und oft ist das auch ausreichend. Manchmal möchten wir aber eine ganz bestimmte Schriftart verwenden und dabei garantieren, dass jeder Nutzer diese auch hat.

Hierzu können wir mittel @font-face eine Schriftart definieren. Wir müssen der Schriftart einen Namen geben, der später bei Auswahl der Schriftart mittels font-family: ... verwendet wird. Zudem müssen wir eine oder mehrere Schriftartendateien angeben, um dem Browser mitzuteilen, wo die Schriftart zu finden ist.

Ein besonders erwähnenswerter Anwendungsfall sind sogenannte Icon-Schriftarten. Dabei handelt es sich um Schriftarten, wo keine Buchstaben enthalten sind, sondern Icons, die wir in unsere Webseite verwenden wollen. Wir könnten zwar auch Bilder direkt für Icons verwenden, eine Icon-Schriftart vereinfacht uns aber die Webentwicklung.

Ein Hinweis noch, wenn wir keine Icon-Schriftarten, sondern "normale" Schriftarten mit Buchstaben (Grapheme) verwenden. Von Buchstaben gibt es verschiedene Variationen, je nach Schriftschnitt (fein, leicht, normal, fett, schwarz) und Schriftlage (kursiv, normal). In der Regel gibt es von gut erstellten Schriftarten für jede Variation eine separate Schriftartendatei. Wir müssen jede dieser Dateien mit jeweils einer @font-face-Regel einbinden und dabei den Schriftschnitt und die Schriftlage setzen.

@font-face {
  font-family: "my-font";
  font-weight: 300;
  font-style: italic;
  src: url("https://example.com/my-font-italic-300.woff2");
}
@font-face {
  font-family: "my-font";
  font-weight: 400;
  font-style: italic;
  src: url("https://example.com/my-font-italic-400.woff2");
}
/* ... */

@media​

Mit der @media-Regel können wir CSS anwenden, je nach Medium, auf dem die Webseite dargestellt wird.

@media screen {
   /* Anzeige auf einem Computer-Bildschirm */
  html {
    color: red;
  }   
}

@media print {
   /* Anzeige beim Drucken über den Browser */
  html {
    color: black;
  }   
}

Weiterhin können wir auch für verschiedene Größen verschiedenes CSS anwenden:

@media (max-width: 319px) {
  /* Styling für Geräte mit kleinem Bildschirm */
}
@media (min-width: 320px) and (max-width: 479px) {
  /* Styling für Geräte mit mittleren Bildschirm */
}
@media (min-width: 480px) and (max-width: 719px) {
  /* Styling für Geräte mit großen Bildschirm */
}
@media (min-width: 720px) {
  /* Styling für Geräte mit sehr großen Bildschirm */
}

Sandbox-Beispiel für Media-Query mit Größen.

Wie im Beispiel oben zu sehen, wird das oft verwendet, um Webseiten responsive zu gestalten, sodass diese auf allen Geräten bedienbar sind.

Zudem können wir auch noch verschiedene Eigenschaften des Anzeigegeräts abfragen, z.B., ob der Nutzer ein Dark-Theme oder ein Light-Theme möchte:

@media (prefers-color-scheme: dark) {
  /* ... */
}
@media (prefers-color-scheme: light) {
  /* ... */
}

Weitere abfragbare Eigenschaften finden sich bei MDN web docs - @media.

@keyframes​

Mit der @keyframes-Regel können wir Animationen erstellen. Diese besteht außer eine Liste von CSS-Eigenschaften, welche in verschiedenen Phasen einer Animation angewandt werden sollen.

@keyframes rotate {
  0% {
    transform: rotate(0deg);
  }
  20% {
    transform: rotate(180deg);
  }
  100% {
    transform: rotate(360deg);
  }
}

0% bezeichnet dabei den Start der Animation, 50% die Mitte der Animation, 100% das Ende der Animation. Wir können beliebig viele Zwischenschritten mit beliebiger Prozentzahl definieren.

Nachdem wir eine Animation definiert haben, können wir diese mittels der Eigenschaft animation verwenden:

.spinner {
  animation-name: rotate;
  animation-duration: 2s;
  animation-timing-function: linear;
  animation-iteration-count: infinite;
}

Einige wichtige Eigenschaften sind noch:

  • animation-iteration-count - Wie oft die Animation abgespielt werden soll. infinite um die Animation endlose abzuspielen.
  • animation-fill-mode: Welche CSS-Eigenschaften angewandt werden sollen, wenn die Animation nicht am Laufen ist. Entweder forward (Eigenschaften vom Ende der Animation werden angewandt, wenn Animation zu Ende), backward (Eigenschaften der Animation vom Anfang werden angewandt, wenn Animation am Anfang), both oder none.

Detaillierte Informationen sind zu finden unter MDN web docs - animation.

@container​

Wir haben gesehen, dass wir mittels der @media-Regel verschiedenes CSS abhängig von der Bildschirmbreite anwenden können.

Manchmal ist es aber nicht die Bildschirmbreite, welche entscheiden ist, sondern der Container, in dem sich ein Element befindet. Nehmen wir beispielsweise an, wir haben eine Tabellenkomponente entwickelt, die andere in ihre Website einbinden können. Wo die Tabelle eingebunden wird, können wir nicht vorhersagen. Die Tabelle könnte etwa im rechten unteren Teil der Webseite eingebunden sein. Wir würden nun gerne abhängig davon, wie viel Platz unserer Tabelle zur Verfügung steht, das Layout der Tabelle beeinflussen.

Dies ist uns möglich via der @container-Regel.

Um von dieser Funktionalität Gebrauch zu machen, müssen wir diese zuerst über die CSS-Eigenschaft container: inline-size aktivieren. Danach können eine entsprechende Container-Query starten:

@container (max-width: 500px) {
  /* Schriftfarbe rot, wenn Containerbreite 500px oder weniger ist. */
  table {
    color: red;
  }
}

Statt container: inline-size könnten wir auch container: size verwenden. container: inline-size reagiert nur auf Änderungen der Breite des Containers, container: size auch auf Änderungen der Höhen.

info

Bei vertikalen Schreibrichtung reagiert container: inline-size nur auf die Höhe und container: size auf Höhe und Breite.

@scope / @layer​

warnung

Vorher bitte unbedingt die Browser-Kompatibilität prüfen!

Zwei recht neue Funktionalitäten im CSS sind Layer und Scope. Diese ermöglichen uns, genauer zu bestimmen, wo unser CSS angewandt werden soll.

Scope

@scope ermöglicht uns, alle Regeln nur auf einen Teil des HTML anzuwenden. Das vermeidet, dass unser CSS unbeabsichtigt anderes CSS aufgrund der Kaskadierung überschreibt. Im folgenden Beispiel werden die Regeln nur auf eine Card-Komponente angewandt. Im Container mit der Klasse card-content könnte Inhalt stehen, den der Nutzer unserer Komponente bereitgestellt hat. Durch die Limitierung über den Scope können wir uns sicher sein, dass unser CSS keine unbeabsichtigten Nebenwirkungen hat.

Layer

@layer ermöglicht uns, eine ganze neue Prioritätsstufe in die CSS-Kaskade einzufügen. Jeder Layer stellt eine separate Stufe dar, der letzte deklarierte Layer hat die höchste Priorität.
