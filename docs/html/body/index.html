Abschnitt 3 - Inhalt (Body)

Im <body> steht der eigentliche Inhalt der Webseite. Die meisten Elemente im <body> entsprechen einem Element, welches der Nutzer auf der Webseite sieht. Wir können mehrere Elemente im Body haben, diese werden vom Browser der Reihe nach dargestellt, von oben nach unten.

HTML definiert viele Elemente, die wir verwenden können. Da HTML eine Living Standard ist, kommen auch nicht selten neue Elemente dazu. In diesem Abschnitt verschaffen wir uns einen Überblick über die wichtigsten Elemente.

Content Model​

Bevor wir uns konkrete Elemente anschauen, noch eine Anmerkung zum sogenannten Content Model in HTML. In XML und HTML können wir innerhalb jedes Elements Kinderelemente einfügen. HTML macht nun bestimmte Einschränkungen, welche Elemente innerhalb von welchen Elternelementen verwendet werden dürfen. Dies nennt der Standard das sogenannte Content Model.

Jedes Element ist in eine oder mehrere Kategorien gruppiert. Einige Kategorien überschneiden sich. Dies ist in folgender Abbildung dargestellt (mit der Maus darüber fahren, um die Elemente der Kategorie zu sehen):

Für jede Art von Element definiert der HTML-Standard dann, welche Inhalte möglich sind. Beispielsweise sagt der Standard zum <label>-Element:

Content model:

Phrasing content, but with no descendant labelable elements unless it is the element's labeled control, and no descendant label elements.

Es ist also zuerst einmal nur "Phrasing Content" erlaubt. Das bedeutet beispielsweise, wir dürfen keine <div>-Elemente (Flow Content) oder <article>-Elemente (Sectioning Content) verwenden. Das folgende HTML ist also invalid:

<label for="email"><div>Ihre E-Mail</div><div>(kein Spam!)</div></label>
<input id="email" name="email" type="input">

Die meisten Browser werden solches invalides HTML trotzdem darstellen und in der Regel nicht einmal eine Warnmeldung uns Entwicklern anzeigen. Es kann uns daher schnell passieren, dass wir unbewusst HTML schreiben, welches streng genommen nicht dem Standard entspricht. Wir sollten immer darauf achten, gültiges HTML zu schreiben, damit jede Art von Software mit unserer Webseite korrekt umgehen kann (Browser, Screenreader, Braille-Display etc).

Von W3C gibt es ein bereitgestelltes Online-Tools, mit dem wir unsere Webseite prüfen können:

https://validator.w3.org/

<div> und <span> werden oft verwendet für eigene interaktive Inhalte und Widgets, wie etwa Suchleisten oder Select-Dropdowns.

<div> wird standardmäßig als Block-Element dargestellt, welches die gesamte Bildschirmbreite einnimmt und beliebigen Flow-Inhalt aufnehmen kann, also fast alle HTML-Elemente. <span> ist ein Inline-Element, welches nur so breit ist wie sein Inhalt. <span> darf nur Phrasing-Inhalt enthalten, also etwa keine <div> oder <article>.

Links​

Links sind das Herzstück des weltweiten Webs. Wir können mit Links auf andere Seite verweisen, entweder innerhalb unserer Webseite oder auf andere Webseiten. Für Links nutzen wir das <a>-Element (a = anchor).

In Sandbox öffnen
a
loading...

Das wichtigste Attribute ist href. Damit geben wir die Zielseite an, auf die verlinkt werden soll. Oft verlinken wir hier entweder eine externe Seite wie z.B. http://example.com oder eine interne Seite wie ../sub-page.html.

Eine URL, auf die verlinkt wird, kann entweder absolut oder relativ sein.

Eine absolute URL ist etwa http://example.com/web/content/index.html oder http://example.com/web/images/logo.png.

Eine relative URL ist nur sinnvoll relativ zu einer Basis-URL. Wenn wir uns zum Beispiel auf der Webseite http://example.com/web/content/index.html befinden, könnte diese Webseite den relativen Link ../images/logo.png enthalten. Die 2 Punkte bedeuten: gehe 1 Verzeichnis nach oben. Danach wird in das Verzeichnis images gegangen und auf die Datei logo.png zugegriffen.

Schließlich können wir auch noch ein sogenannten Fragment verwenden, um auf einen bestimmten Abschnitt innerhalb eines HTML-Dokuments zu verlinken. Das Fragment muss dabei gleich dem id-Attribut des HTML-Elements sein, auf das verlinkt werden soll. Zum Beispiel:

Mit dem target-Attribut steuern wir, wo der Link geöffnet werden soll. standardmäßig wird der Link im aktuellen Fenster oder Tab geöffnet. Der spezielle Wert _blank sorgt dafür, dass der Link sich in einem neuen Tab oder Fenster öffnet. Wir können auch die ID eines Frames als target angeben, dies wird aber eher selten verwendet.

Weiterhin können wir noch das Attribut rel (relationship) setzen, also die Beziehung zur verlinkten Seite, etwa privacy-policy (Datenschutz) oder search (Seite für Suche im Inhalt der Webseite). Oft wird aus Sicherheits- und Performance-Gründen noreferrer noopener gesetzt. Weitere erlaubte Werte sind bei MDN web docs zu finden.

Semantisches HTML​

Semantic HTML ist ein allgemeiner Begriff für HTML-Elemente mit einer bestimmten Semantik (Bedeutung). Beispielsweise steht <h1> für eine Hauptüberschrift. Unabhängig davon, wie die Überschrift per CSS dargestellt wird, kann Software wie Screenreader erkennen, dass es sich um eine Überschrift handelt.

Das grundlegende Element im HTML, um ein Bild anzuzeigen, ist das <img>-Element:

Wir sollten uns an allgemeine Richtlinien für Bilder halten. Die Auflösung sollte so hoch wie nötig, so gering wie möglich sein. Für reale Bilder ist das Bildformat JPG empfehlenswert, für Zeichnungen PNG. WEBP ist ein Format, welches oft in kleineren Dateigrößen resultiert und langsam weitere Verbreitung findet.

Um den Bildinhalt auch für Screenreader verständlich zu machen, sollten wir immer eine aussagekräftige Beschreibung im alt-Attribut hinterlegen. Dieses wird auch angezeigt, wenn das Bild nicht geladen werden konnte.

Das srcset erlaubt uns, höher aufgelöste Bilder anzuzeigen, wenn ein hochauflösendes Gerät verwendet wird:

Findet ein Browser einer dieser Bedingungen als passend, wir das Bild von dem jeweiligen <source> angezeigt. Ansonsten wird das <img> angezeigt.

SVG

Neben Formaten wie PNG und JPEG gibt es noch ein weiteres wichtiges Format: SVG. SVG steht für Scalable Vector Graphics.

Bei pixelbasierten Formaten wie PNG oder JPG wird ein roter Kreis als Menge von Pixeln repräsentiert, wobei Pixel innerhalb des Kreises rot sind und außerhalb des Kreises weiß:

Das Problem dabei ist, dass bei ausreichender Vergrößerung der Rand des Kreises nicht mehr glatt erscheint. Zudem wird für jedes Pixel Speicherplatz gebraucht — obwohl es doch eigentlich nur wenige wichtige Informationen gibt: Ein Kreis an einer Position, mit der Farbe rot.

Bei SVG können wir diese relevante Information direkt ausdrücken:

<svg width="300" height="200" xmlns="http://www.w3.org/2000/svg">
  <circle cx="150" cy="100" fill="red" r="80" />
</svg>

Egal, wie sehr wir an das SVG heranzoomen, die Ränder bleiben immer glatt. SVG werden unter anderem gerne für Logos und Icons verwendet.

Listen​

In Texten kommen öfters auch Listen oder Aufzählungen vor. HTML stellt uns drei verschiedene Arten von Listen bereit:

  • ul (unordered list) — Liste, wo die Reihenfolge der Elemente unwichtig ist.
  • ol (ordered list) — Liste, wo die Reihenfolge der Elemente wichtig ist.
  • dl (definition list) — Liste von Key-Value-Paaren, wo die Value eine Erklärung zum Key ist.

Sowohl bei <ul> als auch bei <ol> geben wir die Listen-Element mit <li> (list item) an. Bei <dl> verwenden wir <dt> (definition term) für den zu erklärenden Key und <dd> (definition description) für die Erklärung.

Tabellen​

Tabellen dienen, wie der Name sagt, für tabellarische Daten. Wir sollten Tabellen auch nur dafür verwenden, nicht als Hilfsmitteln zum Layout. Früher wurde dies oft getan, da es kaum bessere Möglichkeiten gab. Inzwischen gibt CSS uns wesentlich bessere Möglichkeiten, das Layout unserer Seite gemäß unseren Wünschen zu gestalten.

hinweis

Tatsächlich ist es aus Entwicklersicht eher schwieriger, Tabellen schön und gut darzustellen und zu gestalten. Auch aus Sicht der Responsiveness (Darstellung auf Geräten mit kleiner Bildschirmbreite) müssen wir bei Tabellen entsprechend Testen und Vorkehrungen treffen.

Das Hauptelement der Tabelle ist <table>. Der Tabellenkopf (Kopfspalte) wird mit <thead> angegeben, der eigentliche Inhalt mit <tbody>. Für jede Zeile schreiben wir ein <tr> (table row), für jede Zelle in der Zeile ein <td> (table data). Der Tabellenkopf hat meist nur eine Zeile, die Spalten werden <th> statt <td> angegeben.

Playground — Tabelle

Im oberen Beispiel sehen wir auch noch das <caption>-Element für die Überschrift der Tabelle und das <tfoot>-Element für die Fußzeile der Tabelle.

Zudem wird an der 1. Spalte der Fußzeile <td colspan="2"> verwendet. Damit können wir erreichen, dass eine Zelle sich über mehrere Spalten erstreckt. Analog dazu gibt es auch das Attribut rowspan, wenn eine Zelle mehrere Zeilen einnehmen soll.

Formulare​

Formulare werden verwendet, um Daten zu erfassen. Das Onlinezugangsgesetz (OZG) verpflichtet Behörden dazu, Verwaltungsleistungen auch digital über Verwaltungsportale anzubieten.

Webformular sind hierzu ein geeignetes Mittel, um etwa Anträge online zu ermöglichen.

Das Hauptelement, dass wir benötigen, um Webformular umzusetzen, ist das <form>-Element. Innerhalb eines <form> können wir verschiedene Eingabefelder oder Buttons zum Absenden platzieren. Zur Illustration hier ein einfaches Formular, wo der Vor- und Nachname eingegeben werden muss:

In Sandbox öffnen
form-simple
loading...

Das <form>-Element hat noch einige andere Attribute, das wichtigste ist aber action. Hier geben wir die URL an, an welche die eingegebenen Daten beim Absenden gesendet werden.

info

Die Sandbox hat eine spezielle Vorkehrung für Formulare. Beim Absenden wird das eigentliche Submit blockiert, stattdessen werden die Daten angezeigt, die gesendet werden würden. Somit können Formulare auf einfache Weise ohne ein action-Attribut getestet werden.

Im Folgenden schauen wir uns einige typische Inhalte von Formularen an.

Eingabe-Elemente​

Mit Eingabe-Elementen erfassen wir Daten vom Nutzer. Jedes Eingabefeld sollte einen technischen Namen besitzen, den wir mit dem Attribut name angeben. Der name bestimmt, unter welchem Schlüssel der Wert des Elements beim Absenden an den Server übertragen wird.

Auch sollte jedes Eingabefeld ein <label> mit einem Beschriftungstext für das Feld haben. Das Label ist wichtig, damit etwa Screenreader wissen, was der Zweck des Elements ist. Das <label> muss ein for-Attribut haben, dessen Wert der ID des Eingabe-Elements entspricht:

In Sandbox öffnen
form-label
loading...

Verwenden wir ein <label> mit einem korrekten for-Attribut, kann der Nutzer auch auf das Label klicken, um das Eingabe-Element zu fokussieren.

input

<input> ist ein einzeiliges Eingabefeld und das wohl am meisten verwendete Formularelement.

Mit dem Attribut type können wir auch verschiedene Arten von Eingaben ermöglichen, beispielsweise type="color" für einen Farbwähler oder type="date" für einen Datumswähler. Weitere Typen bei MDN web docs.

Das Attribut required können wir setzen, um ein Element als Pflichtfeld zu markieren. Der Browser verbietet dann das Absenden, wenn kein Wert eingetragen wurde.

Der Nutzer könnte etwa einfach über die Dev-Tools die Validierungsregeln entfernen. Falls erforderlich, muss der Server, der die Daten entgegennimmt, entsprechende Validierungen durchführen!

Schließlich gibt es noch <input> vom type="hidden". Solche Eingabefelder sind nicht für den Nutzer sichtbar und können in der Regel nicht durch diesen bearbeitet werden. Hidden-Inputs werden gerne in Kombination mit JavaScript genutzt, um berechnete oder anderweitig abgeleitete Daten beim Absenden mit zu übertragen.

Ein anderer Anwendungsfall sind Widgets, also per JavaScript geschriebene Elemente, die mehr Funktionalität bieten als Elemente, die der Browser bereitstellt. Hier können wir dann ein Hidden-Input verwenden, um die Daten zu übertragen.

Button​

Wir haben bereits das <button>-Element gesehen. Ein Button ist eine klickbare Schaltfläche. Die Art des Button geben wir mit dem type-Attribut an. Im HTML gibt es 3 Arten von Buttons:

  • submit — Sendet das Formular ab.
  • reset — Setzt alle Eingabefelder auf den ursprünglichen Wert zurück.
  • button — Führt keine Aktion beim Klick durch.

type="button" können wir verwenden, um eigene Aktionen beim Klick durchzuführen, in dem wir entsprechendes JavaScript schreiben.

Den Name des Buttons geben wir mit dem Attribut name an. Über dieses Namen können wir beim Absenden herausfinden, welcher Button gedrückt wurde.

In Sandbox öffnen
form-buttons
loading...

An dieser Stelle noch ein Hinweis. Drückt der Nutzer in einem Formularfeld auf Entern, ist das Standardverhalten von Browsern, so vorzugehen, als ob der Submit-Button gedrückt wurde. Es wird also das Formular abgesendet. Bei kleineren Formularen wie Suchmasken kann das sinnvoll sein. Bei längeren Formularen sorgt das aber schnell dafür, dass Nutzer unbeabsichtigt Formulare absenden.

Um das zu verhindern, können wir entweder mittels JavaScript das Absenden verhindern. Einfacher ist, ganz am Anfang des Formulars einen versteckten Submit-Button einzuführen, der das Absenden verhindert. Das funktioniert deshalb, weil der Browser beim Klick auf Enter den ersten Submit-Button auslöst.

Iframe​

warnung

Früher gab es auch das <frame>-Element, welches man auch für das Layout einer Seite nutzen konnte. <frame> ist obsolet, und wird inzwischen nicht mehr verwendet.

Mit dem <iframe>-Element (inline frame) können wir eine andere Webseite in unsere Webseite einbinden.

<iframe src="http://example.com/map-widget.html"></iframe>

Der Vorteil von einem <iframe> ist, dass dessen Inhalt vollständig vom Rest der Webseite abgekapselt ist. Das betrifft sowohl das Design und Layout (CSS) als auch JavaScript-Logik. Somit ist sichergestellt, dass die beiden verschiedenen Seiten sich nicht gegenseitig beeinflussen.

Etwa verwendet unsere Sandbox einen <iframe>, um das eingegebene HTML darzustellen. Somit ist garantiert, dass die Sandbox selber nicht durch die eingegebenen Inhalt kaputt gemacht wird.

Allgemein werden <iframe> gerne für die Einbindung von Widgets von Drittanbietern verwendet. Hier ist ein Beispiel von mdr.de, wo ein <iframe> genutzt wird, um eine interaktive Grafik zu Autodiebstählen vom Anbieter datawrapper.de einzubinden.
