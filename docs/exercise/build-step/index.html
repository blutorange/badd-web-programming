Aufgaben zum Build-Step

Diese Aufgaben betreffen den Build-Step, also wie wir Quellcode in die fertigen JavaScript-, CSS- und HTML-Dateien überführen.

Wir werden im folgenden Node.js als serverseitige JavaScript-Runtime verwenden. Wer möchte, darf aber gerne auch Alternativen wie Deno oder Bun ausprobieren.

Es empfiehlt sich, die Aufgaben der Reihe nach auszuführen, da diese aufeinander aufbauen.

info

Ein Demoprojekt, welches diese Build-Schritte illustriert, ist im git zu finden unter packages/build-step.

Aufgabe 1 - Hallo Welt​

Installieren Sie Node.js in einer geeigneten Form und führen Sie über dessen REPL den folgenden JavaScript-Code aus!

console.log(`Hallo Welt! 1+2=${1+2}`)

Hinweise:

  • Ich empfehle die aktuelle LTS-Version.
  • Unter Linux ist der NVM - node version manager sehr praktisch. Er lässt sich mit einem Bash-Befehl installieren und erlaubt den einfachen Wechseln zwischen Node-Versionen.
  • Unter MacOS soll nvm auch funktionieren, habe ich aber selber noch nicht probiert. Es gibt auch ein Homebrew-Package für nvm und Node.js

Aufgabe 2 - Script mit Node.js ausführen​

Legen Sie folgende Dateien an und führen Sie anschließend die Datei bin/compare.js mittels Node.js aus! Was macht das Script?

Datei bin/compare.js
const fs = require("node:fs/promises");
const { setDiff, setIntersect } = require("../src/js/set-utils.js");

async function main() {
  if (process.argv.length !== 4) {
    console.log(`Usage: ${process.argv[0]} ${process.argv[1]} <source> <target>`);
    console.log("  Compares the contents of the files in the source and target folder");
    return;
  }

  const sourceFolder = process.argv[2];
  const targetFolder = process.argv[3];

  const sourceContents = await fs.readdir(sourceFolder, { recursive: true });
  const targetContents = await fs.readdir(targetFolder, { recursive: true });

  const sourceSet = new Set(sourceContents);
  const targetSet = new Set(targetContents);

  const onlyInSource = setDiff(sourceSet, targetSet);
  const onlyInTarget = setDiff(targetSet, sourceSet);
  const commonFiles = setIntersect(sourceSet, targetSet);

  if (onlyInSource.size > 0) {
    console.log("Files only in source that do not exist in target:");
    for (const item of onlyInSource) {
      console.log(`  - ${item}`)
    }
  }

  if (onlyInTarget.size > 0) {
    console.log("Files only in target that do not exist in source:");
    for (const item of onlyInTarget) {
      console.log(`  - ${item}`)
    }
  }

  if (commonFiles.size > 0) {
    console.log("Files that exist in both folders:");
    for (const item of commonFiles) {
      console.log(`  - ${item}`)
    }
  }
}

main().catch(e => {
  console.error("Failed to run script", e);
  process.exit(1);
});
Datei src/js/set-utils.js
/**
 * Computes the set difference a \ b.
 * @param {Set<string>} a First set.
 * @param {Set<string>} b Second set.
 * @returns {Set<string>} b The difference a \ b.
 */
function setDiff(a ,b) {
  const diff = new Set(a);
  b.forEach(x => diff.delete(x));
  return diff;
}

/**
 * Computes the set intersection a ^ b.
 * @param {Set<string>} a First set.
 * @param {Set<string>} b Second set.
 * @returns {Set<string>} b The intersection a ^ b.
 */
function setIntersect(a ,b) {
  const intersect = new Set();
  a.forEach(x => b.has(x) && intersect.add(x));
  return intersect;
}

module.exports.setDiff = setDiff;
module.exports.setIntersect = setIntersect;

Aufgabe 3 - ESM mit Node.js ausführen​

Wandeln Sie das Dateien aus Aufgabe 2 in ESM um!

Sie werden feststellen, dass sich die Dateien dann entweder nicht mehr ausführen lassen oder Warnungen beim Ausführen ausgegeben werden.

Um das zu beheben, legen Sie die Datei package.json an. Neben grundlegenden Eigenschaften wie name,description und version müssen Sie hier "type": "module" eintragen.

Hinweis: ESM verwendet import-Statements statt require und export-Statements statt module.exports.

Aufgabe 4 - Externe Bibliothek einbinden​

Installieren Sie die externe Bibliothek esbuild (Homepage, NPM)! Legen Sie anschließend folgende Dateien an und führen Sie die Datei scripts/build.js aus! Führen Sie nun auch die Datei dist/index.js aus! Was macht das Script?

Datei scripts/build.js
import * as esbuild from "esbuild";

const buildResult = await esbuild.build({
    entryPoints: ["src/js/index.js"],
    outdir: "dist",
    bundle: true,
    target: "es2020",
    minify: true,
    write: true,
    logLevel: "info",
});

console.log(`Build finished with ${buildResult.errors.length} errors, ${buildResult.warnings.length} warnings`);
Datei src/js/index.js
import { factorial } from "./factorial.js";

for (let i = 0n; i <= 20n; i++) {
  console.log(`${i}! = ${factorial(i)}`);
}
Datei src/js/factorial.js
export function factorial(x) {
  return x > 1n ? x * factorial(x - 1n) : 1n;
}

Aufgabe 5 - Script in HTML-Seite einbinden​

Erstellen Sie unter dist/html/index.html eine einfache HTML-Seite und binden Sie dort das Script aus dist/index.js ein! Öffnen sie die HTML-Seite im Browser und bestätigen Sie, dass das Script korrekt funktioniert!

Aufgabe 5 - Lokaler Web-Server​

Für einige Funktionalitäten wie HTTP-Requests per fetch ist es notwendig, die HTML-Seite über einen Webserver per HTTP-Protokoll zu öffnen.

Installieren Sie hierzu die externe Bibliothek http-server! Starten Sie den Server im dist-Ordner und rufen Sie die Webseite im Browser via HTTP auf!

Hinweis 1: Lesen Sie die verlinkte Dokumentation bei NPM!

Hinweis 2: Der http-server hat auch eine Option zum Deaktivieren des Browser-Cache. Dies kann bei der Entwicklung sehr hilfreich sein. Wenn Sie diese aktivieren, wie funktioniert diese? Schauen Sie sich im Netzwerk-Tab die HTTP-Requests an!

Aufgabe 6 - NPM-Script für Server​

Öffnen Sie die package.json und fügen sie hier ein Script ein, sodass Sie anschließend den HTTP-Server mittels npm run start starten können!

Hinweise:

  • Siehe die Doku von NPM bzgl. scripts
  • Innerhalb eines Script stehen NPM-Binaries zur Verfügung, d.h. das npx ... am Anfang kann weggelassen werden.

Aufgabe 6 - NPM-Script für Build​

Ergänzen Sie ein NPM-Script für den Build-Prozess, sodass Sie mittels npm run build die Dateien im dist-Ordner erzeugen können!

Hinweis: Siehe Aufgabe 4.

Aufgabe 7 - Watch Mode​

Ergänzen Sie zuerst das scripts/build.js-Script um einen Watch-Mode! Wenn in der Kommandozeile das --watch mit angegeben wird, soll der Watch-Mode von esbuild aktiviert werden. Fügen Sie nun noch ein Script build-dev ein, was das gleiche tut wie build, aber mit aktivierten Watch-Modus.

Hinweise:

  • Bei einem [Watch-Mode] wird bei jeder Änderung eine Quellcode-Datei der Bundler erneut ausgeführt, sodass die Dateien im dist-Ordner den aktuellen Stand widerspiegeln.
  • Auf die Argument der Kommandozeile können Sie mittels process.argv zugreifen.
  • Lagern Sie die esbuild-Optionen in eine Variable aus und verwenden Sie esbuild.context(...) sowie context.watch() für den Watch-Mode. Siehe auch esbuild - build (auf den JS-Tab klicken).

Aufgabe 8 - Copy-Script für HTML​

Verschieben Sie die Datei dist/html/index.html nach src/html/index.html und ergänzen Sie das Script scripts/build.js um zusätzliche Logik, welche alle HTML-Dateien von src/html nach dist/html kopiert!

Hinweis: Dateizugriff geschieht in Node.js mittels den Modules node:fs/promises und node:path.

Alternativ können Sie auch esbuild zum Kopieren verwenden, indem Sie die HTML-Seiten zu den Entry-Points hinzufügen und als loader für .html den copy-Loader setzen.

Aufgabe 9 - Bundling für CSS​

Erstellen Sie folgende CSS-Dateien im src-Ordner. Binden Sie die src/css/index.css in der src/html/index.html ein.

Ergänzen Sie nun das Script scripts/build.js so, dass analog zum JavaScript eine minifizierte Datei dist/css/index.css erstellt wird!

Datei src/css/index.css
@import url("./variables.css");
@import url("./base.css");
Datei src/css/variables.css
:root {
  --font-size: 20px;
  --theme-color: blue;
}
Datei src/css/base.css
html {
  font-size: var(--font-size);
  color: var(--theme-color);
}

Hinweis: Sie können mehrere Entry-Points definieren.

Aufgabe 10 - Leaflet​

Installieren Sie per NPM die externe Bibliothek leaflet.js (Homepage, NPM)!

Bereiten Sie die HTML-Seite entsprechend vor, indem Sie die benötigten HTML-Elemente erstellen.

Importieren Sie leaflet.js nun über ein Import-Statement in ihre JavaScript-Datei und initialisieren Sie leaflet.js, sodass in der Webseite nun eine interaktive Karte sichtbar ist.

Erstellen Sie einen Button auf der HTML-Seite, der beim Drücken einen Marker auf der Karte anzeigt (Position ist egal).

Hinweis: Siehe auch auch den Quick start guide von leaflet.js.

Aufgabe 13 - Dynamisches Nachladen von Inhalt​

Legen Sie im Ordner src/html einige Unterseiten an, z.B. mit den Namen page1.html, page2.html uws. Diese Unterseiten sollen keine vollständigen HTML-Seiten sein, sondern nur den Inhalt enthalten, der in der <main>-Sektion der src/main/index.html geladen wird.

Fügen Sie nun in der src/index.html Links in der Navigationsleiste auf diese Unterseiten ein. Schreiben Sie anschließend JavaScript, welches beim Klick auf diese Links den Inhalt der entsprechenden Seite per HTTP-Request (AJAX) lädt und die <main>-Sektion der Seite aktualisiert!

Aufgabe 15 - TypeScript (optional)​

Falls Sie TypeScript ausprobieren möchten, fügen Sie TypeScript zu Ihrem Projekt hinzu!

Hinweise:

  • Installieren der externen Bibliothek typescript.
  • Erstellen der tsconfig.json im Root-Verzeichnis. Siehe auch What is a tsconfig.json. Speziell include sollte auf den src-Ordner gesetzt werden. Da wir esbuild zum Bauen nutzen, kann noch das Flag emitDeclarationOnly aktiviert werden.
  • Umbenennen der *.js-Dateien in *.ts.
  • esbuild kann problemlos mit TypeScript umgehen, hier sollte keine Anpassungen erforderlich sein. Nur der EntryPoint muss auf src/index.ts geändert werden.
  • Erstellen eines NPM-Script verify, welches TypeScript ausführt und die Typen prüft.
