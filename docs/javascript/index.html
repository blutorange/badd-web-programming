Kapitel 3 - JavaScript und TypeScript

In diesem Kapitel schauen wir uns die Programmiersprache JavaScript an.

Mit JavaScript können wir Webseiten dynamisch gestalten. Nahezu alle modernen Webseiten verwenden JavaScript in der einen oder anderen Form und ist kaum noch wegzudenken.

  • Wikipedia
    • Optional, aber Zusatzfunktionen wie Bildvorschau im Overlay
  • Webshops
  • Soziale Netzwerke wie Facebook, Twitter, Reddit, Bluesky
    • UI wird teilweise erst durch den Browser erzeugt. Meta (Facebook) war etwa beteiligt an der UI-Bibliothek react.
  • Videoplattformen wie Odysee oder YouTube
  • Anwendungen wie draw.io (Diagramme zeichnen) oder Pixlr (Bildbearbeitung).

JavaScript wird offiziell bezeichnet als EcmaScript.

EcmaScript wird betreut von ecma iInternational, welche jedes Jahr eine neue Version der Programmiersprache herausbringen. Die Spezifikation selber kann hier eingesehen werden: ecma262.

TC39 ist eine von ecma International organisierte Gruppe von Fachleuten, welche sich um die Weiterentwicklung der Programmiersprache kümmern. Neue Features für EcmaScript heißen Proposals, welche durch verschiedene Stages laufen. Schafft ein Feature es in die letzte Stage 4, wird es offiziell in den EcmaScript-Standard aufgenommen. Einige nennenswerte Proposals aktuell sind etwa:

  • Temporal-API
  • Decorators
  • Shadow Realm (Hat nichts mit einem Kartenspiel zu tun!)
  • Type-Annotations
  • Pipeline-Operator

Wer etwa über (oder besser: mit) JavaScript lachen möchte, dem sei folgende Seite empfohlen:

github.com/denysdovhan - What the f*ck JavaScript?
