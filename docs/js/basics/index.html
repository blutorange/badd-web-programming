Abschnitt 1 - Grundlagen

JavaScript ist eine Multi-Paradigma-Programmiersprache. Das heißt grob, wir können damit auf verschiedene Weisen programmieren. Datentypen​

JavaScript wird oft als untypisierte Sprache bezeichnet. Das ist nicht ganz korrekt, es gibt verschiedene Datentypen, die teilweise auch beim Ausführen des Programms geprüft werden. Folgende Datentypen gibt es in JavaScript:

// Fehlen eines Werts
undefined;

// Wahrheitswerte
false, true;

// Zahlen (technisch: immer 64-bit Float)
2, -5, 9.42, 123_456;

// Präsize, beliebig große Ganzzahlen (bigint)
900_719_925_474_099_132n;

// Vergleiche auch das Ergebnis von
900_719_925_474_099_132n + 2n;
900_719_925_474_099_132 + 2;

// Zeichenketten (Strings)
"", "Hallo Welt", `1 + 2 = ${1 + 2}`;

// Symbole
Symbol.iterator, Symbol("custom");

// Funktionen
function sum(x ,y) { return x + y }
const multiply = (x, y) => x * y; 

// Objekte
{name: "Mary", age: 18}, [1, 10, 100], /\d+/, null;

Und zwar undefined, weil undefined in JavaScript immer auftreten kann, während null explizit verwendet werden muss.

Objekte​

Objekte sind ein fundamental wichtiger Bestandteil von JavaScript. Daher möchten wir uns diese nochmal genauer anschauen.

Das grundlegende Merkmal von Objekten ist, dass diese Eigenschaften und Methoden haben, welche verwendet werden können. Hierzu wird nach dem Objekt ein Punkt verwendet:

({name: "Johanna"}).name; // "Johanna"
"foo".length; // 3

(255).toString(16); // "ff"
"foo bar".substring(3); // "foo"
[1, 2, 3].includes(2); // true

Alles außer undefined und null hat solche Eigenschaften, auch wenn deren typeof nicht object ist.

Variablen​

Programme arbeiten mit Daten, diese werden in Variablen gehalten, um im Code darauf verweisen zu können.

const x = 10;
const y = x + 5;
for (let i = 0 ; i < y; i += 1) {
  console.log(i);
}

Was ist der Unterschied zwischen const und let?

  • const bedeutet, der Wert muss direkt zugewiesen werden und kann sich nicht mehr ändern.
  • letbedeutet, der Wert der Variable kann beliebig oft verändert werden.

In der Praxis ist es üblich, Variablen nicht ständig zu ändern. Dies erhöht das Verständnis und die Lesbarkeit des Codes. let wird nur verwendet, wenn es erforderlich ist, wie etwa bei einer For-Schleife.

Variablen sind immer nur in dem Block gültig, wo sie deklariert wurden:

Anweisungen und Ausdrücke​

Ein Ausdruck ist eine Berechnung mit Werten, die in einem neuen Wert resultiert.

3 + 4

Ausdrücke können geschachtelt werden, Klammern bei Bedarf zur Reihenfolge:

5 * ( 4 + 3 ) / Math.cos(2)

Eine Anweisung ist ein Teil eines Programms, die einen bestimmten Zweck erfüllt und mit einem Semikolon ; abgeschlossen wird. Anweisungen enthalten oft Ausdrücke.

Ein Beispiel für eine Anweisung ist ein Funktionsaufruf:

console.log(1);

Um mehrere Anweisungen nacheinander auszuführen, wird eine Block-Anweisung verwendet:

{
  console.log(1);
  console.log(2);
  console.log(3);
}

if (4 ** 5 > 5 ** 4) console.log(`4**5 = ${4 ** 5} > 5**4 = ${5 ** 4}`);

Vergleichsoperationen​

Wir können ganz normal < und > sowie <= und >= verwenden, um Zahlen und Strings zu vergleichen.

Mit den Operatoren === und !== können wir prüfen, ob 2 Werte gleich oder ungleich sind.

Historisch gibt es noch die Operatoren == und !=. Diese werden nur noch selten oder gar nicht mehr verwendet, da ihnen komplexe Regeln unterliegen, wie Werte miteinander verglichen werden. Die Regeln für === sind hingegen wesentlich einfacher. So einfach, dass wir uns diese Regeln hier anschauen können. Um zwei Werte x und y mittels === zu vergleichen:

  1. Ist typeof x !== typeof y, sind die Werte ungleich.
  2. Sind x und y vom Typ object, gibt true zurück, wenn es exakt die gleichen Objekte sind (reference-equal), sonst false.
  3. Andernfalls gib true zurück, wenn der Inhalt der Werte gleich ist (equal-by-value), sonst false.

Funktionen​

Funktionen sind ein wichtiges Werkzeug, um Code in wiederverwertbare Bestandteile zu gruppieren. Eine Funktion kappselt einen bestimmten Code-Teil. Sie hat eine beliebige Anzahl an Parametern, die übergeben werden können, und genau einen Rückgabewert.

Parameter, für die wir keinen Wert übergeben, werden durch JavaScript auf undefined gesetzt. Geben wir keinen Wert mittels return zurück, gibt die Funktion automatisch undefined zurück.

Default-Werte für Parameter können durch = angegeben werden.

Prototypen​

JavaScript basiert auf dem Konzept, dass jedes Objekt einen Prototyp hat (welcher ebenfalls wieder einene Prototypen haben kann usw.). Die Menge aller Eigenschaften eines Objekts ergibt sich aus der Menge der Eigenschaften aller Prototypen.

Die direkte Verwendung von Prototypen kann schnell dazu führen, unübersichtlichen Code zu schreiben und werden daher nicht mehr so häufig verwendet. Auf eine genauer Erklärung zu Prototypen möchte ich daher an dieser Stelle verzichten. Weitere Informationen finden sich etwa bei MDN.
