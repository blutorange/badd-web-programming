Abschnitt 7 - Fetch API

Bei der Programmierung von Webseiten müssen wir auch öfters per JavaScript HTTP-Requests ausführen, um dynamische Inhalte nachzuladen.

Historisch hat sich die Bezeichnung AJAX für HTTP-Request eingebürgert. AJAX steht für

Asynchronous Javascript And eXtensible markup language

Dies rührt daher, dass ursprünglich XML-Dokumente damit abgeholt werden sollten.

Um dies zu tun, können wir die Fetch-API verwenden, welche im Wesentlichen aus nur einer Funktion fetch besteht. Die Spezifikation findet sich unter Fetch - Living Standard.

In Sandbox öffnen
fetch-simple
loading...

Wie wir sehen können, verwendet fetch Promises, da HTTP-Request asynchron sind.

Im folgenden wollen wir uns die fetch-Funktion noch etwas genauer anschauen.

note

Historisch gab es noch ein XMLHttpRequest, mit dem auch HTTP-Requests ausgeführt werden können. Dieses ist aber wesentlich schwerer zu bedienen (einer Sage nach finden es Leute schon schwer, sich zu merken, wie es überhaupt geschrieben wird).

Erstellen eines Requests​

Ein HTTP-Request besteht aus 4 Teilen: Einer Zieladresse, einem HTTP-Verb, einer Liste von Request-Headern und einem Request-Body.

Die Zieladresse übergeben wir wie bereits gesehen als ersten Parameter. Als zweiten Parameter akzeptiert die fetch-Funktion noch ein Objekt mit weiteren Konfigurationsmöglichkeiten.

Die Angabe des HTTP-Verbs geschieht über die Eigenschaft method, die Angabe der Request-Header über headers und die Angabe des Request-Body über body:

In Sandbox öffnen
fetch-post
loading...
info

Achtung: Aus Sicherheitsgründen sind nicht alle HTTP-Header erlaubt. Server können bei Bedarf über den Response-Header Access-Control-Allow-Headers zusätzliche Header erlauben.

Weitere Eigenschaften, die beim Erstellen eines Requests übergeben werden können, finden sich in der Dokumentation von RequestInit.

Erwähnenswert ist vielleicht noch die Eigenschaft credentials. Wenn diese gesetzt ist, werden Cookie mitgesendet. Dies ist notwendig, falls der Zugriff etwas über einen Session-Cookie geschützt ist.

Auswerten einer Response​

Nachdem wir ein Request vorbereitet und abgesendet haben, müssen wir die Response auswerten. Die fetch-Methode liefert ein Response-Objekt zurück. Dieses hat verschiedene Eigenschaften zum Zugriff auf die Response:

  • status - Der Statuscode der Response, z.B. 200.
  • headers - Die Response-Header, welche vom Server zurückgeliefert wurden.
  • text() - Methode zum Zugriff auf den Response-Body als Text.
  • json() - Effektiv das gleiche wie das Aufrufen von JSON.parse auf text().
  • blob() - Methode zum Zugriff auf den Response-Body als Blob.
