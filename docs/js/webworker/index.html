Abschnitt 8 - Web Worker

JavaScript wurde traditionell für kleine Dinge eingesetzt, um Webseiten etwas mehr Dynamic zu verleihen, etwa der Vergrößerung eines Bilds beim Überfahren mit der Maus.

Inzwischen werden auch komplexe Anwendungen als Webanwendung umgesetzt. Diese erfordern teilweise Berechnungen, welche länger dauern, beispielsweise:

  • Der Code-Editor in der Sandbox (monaco-editor) muss Code-Vorschläge und Fehler in der Syntax berechnen.
  • Office 365 / Word hat einen Webworker für Telemetry.
  • PixiJS, eine bekannte HTML5-Game-Engine, unterstützt WebWorker, um Spiel-Logik und Berechnungen dort auszuführen.

Würden wir diese längeren Berechnungen ohne WebWorker ausführen, kann das zur Folge haben, dass eine Webseite eine Zeit lang unbedienbar wird oder stockend wirkt (unresponsive).

Bei Verwendung eines Web-Worker müssen wir bedenken, dass dieser in der Regel vom Browser in einem separaten Thread ausgeführt und losgelöst vom Main-Thread ist. Als Folge davon steht die DOM-API nicht zur Verfügung und es kann nicht direkt auf HTML-Elemente oder Funktionen aus dem Main-Thread zugegriffen werden.

Prinzipiell ist ein Web-Worker nur eine JavaScript-Datei, welche auf eine spezielle Art geladen wird:

// Starten des Workers
const worker = new Worker("https://example.com/my-worker.js");

// ...

// Beenden des Workers
worker.terminate();

Um mit dem Worker zu kommunizieren und Daten auszutauschen, müssen sogenannte Message verwendet werden. Eine Message ist lediglich nur ein JavaScript-Wert, der an den Worker übertragen werden soll.

worker.postMessage({type: "computePlayerPathThroughLabyrinth"});

Um im Worker-JavaScript Message zu empfangen, müssen wir einen Event-Listener registrieren.

globalThis.addEventListener("message", event => {
  console.log(event.data);
});

Analog können wir einen Event-Listener auf den Worker registrieren, um Message von diesem zu empfangen:

worker.addEventListener("message", event => {
  console.log(event.data);
});

Hier findet sich noch ein Beispiel in der Sandbox, wo ein Worker verwendet wird, um Primzahlen zu berechnen. Durch den Worker bleibt die Seite responsive:

Worker-Beispiel mit Primzahlenberechnung

info

Beim Senden von Message wird der sogenannte strukturelle Klon-Mechanismus auf das gesendete Objekte angewandt. Vereinfacht gesagt bedeutet das, dass eine Kopie von dem Objekt erstellt wird, sodass die beiden Seiten getrennte Kopien des Objekts besitzen.

const data = {type: "computePlayerPathThroughLabyrinth"};
worker.postMessage(data);
data.hint = "Look backwards";

Die Änderungen des data-Objekte nach dem Senden der Nachricht hat keine Auswirkung mehr auf die Daten, welcher der Worker empfängt, da eine Kopie von dem Objekt erstellt wurde.
