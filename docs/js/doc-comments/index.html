Abschnitt 9 - Dokumentation

Zu guten Programmieren gehört nicht nur, funktionsfähigen Code zu schreiben, sondern diesen auch zu dokumentieren. Damit helfen wir anderen Entwicklern, die den Code sich anschauen oder damit arbeiten müssen.

Ein Mittel zur Dokumentation besteht darin, direkt im Code an Funktionen und Klassen zu beschreiben, was diese tun und was dabei beachtet werden muss.

info

Doc-Kommentare gibt es auch in vielen anderen Programmiersprachen, wie in Java, in C#, in Haskell oder in Python.

In JavaScript gibt es dafür eine spezielle Art von Kommentaren, sogenannte Doc-Kommentare. Dabei wir ein Multi-Line-Kommentar mit der Sternen am Anfang verwendet:

/**
 * Checks if a `value` starts with a given `prefix`. __Both values must be
 * strings.__
 * 
 * If the prefix is the [empty string](https://en.wikipedia.org/wiki/Empty_string),
 * always returns `true`.
 * 
 * If the value is the empty string, always returns `false`, unless the prefix
 * is also the empty string.
 * 
 * @example
 * ```js
 * startsWith("foo-bar", "foo-") // => true
 * startsWith("foo-bar", "-bar") // => false
 * startsWith("foo", "") // => true
 * startsWith("", "foo") // => false
 * startsWith("", "")    // => true
 * ```
 * 
 * @param {string} value - A string to check whether it starts with a prefix,
 * e.g. `foo-bar`.
 * @param {string} prefix - A prefix to test, e.g. `foo-`.
 * @returns {boolean} `true` if the value starts with the given prefix, `false`
 * otherwise.
 */
function startsWith(value, prefix) {
  if (prefix.length === 0) {
    return true;
  }
  if (value.length === 0) {
    return false;
  }
  return value.substring(0, prefix.length) === prefix;
}

Solche Kommentare werden von den meisten IDE und Code-Editoren besonders behandelt und angezeigt:

Ein Doc-Kommentar ist immer aus 2 Abschnitten aufgebaut:

  • Einer Freitextbeschreibung
  • Eine Liste von Tags, die mit @ beginnen und strukturiert spezifische Eigenschaften beschreiben.

Die häufigsten Tags sind @param und @returns, welche bei Funktionen verwendet werden, um die Parameter und den Rückgabewert zu dokumentieren.

Innerhalb geschweifter Klammern wird dabei der Typ des Arguments angegeben. Weitere Typen lernen wir im nächsten Abschnitt kennen.

Innerhalb der Doc-Kommentar kann Text auch ausgezeichnet werden (Links, fette Schrift, Code-Beispiele, etc.). Hierzu wird Markdown verwendet.

Markdown ist eine Auszeichnungssprache, welche versucht, gleichzeitig auch gut in Textform lesbar und bearbeitbar zu sein. Auch diese Tutorialseiten sind in Markdown geschrieben:

  • Mittels einfachen Unterstrichen _kursiv_ wird Text kursiv gedruckt.
  • Mittels doppelten Unterstrichen __fett__ wird Text fett gedruckt.
  • Mittels Backticks `1+2` wird Text als Code markiert: 1+2
  • Mittels Klammern [example](https://example.com) wird ein Link example eingefügt.
  • Mittels einem Strich am Anfang - Anführungspunkt wird eine Liste wie diese erzeugt.
  • Mittels 3 Backticks wird ein Code-Schnipsel eingefügt.
```js
console.log("Hallo Welt");
```
